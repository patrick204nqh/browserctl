# Implementation Plan — browserctl v0.4 Hardening

- [ ] 1. Ruby version pin
  
  - [ ] 1.1 Update gemspec
    - Change `required_ruby_version = ">= 3.2"` → `">= 3.3"` in `browserctl.gemspec:24`
    - _Requirements: REQ-1_

- [ ] 2. Secure recording file permissions
  
  - [ ] 2.1 Harden `Recording.start`
    - Pass `mode: 0o700` to `FileUtils.mkdir_p(RECORDINGS_DIR, ...)` in `lib/browserctl/recording.rb`
    - After `FileUtils.rm_f(log_path(name))`, add `FileUtils.touch` + `File.chmod(0o600, ...)` to pre-create the file with restricted permissions
    - _Requirements: REQ-2_

- [ ] 3. Cross-step state in WorkflowContext
  
  - [ ] 3.1 Add `store`/`fetch` to `WorkflowContext`
    - Add `@_store = {}` to `WorkflowContext#initialize` in `lib/browserctl/workflow.rb`
    - Add `def store(key, value)` and `def fetch(key)` public methods
    - `fetch` raises `KeyError` with descriptive message on missing key
    - _Requirements: REQ-3_
  
  - [ ] 3.2 Add spec coverage
    - Add tests to `spec/unit/workflow_spec.rb` for `store`/`fetch` happy path and missing-key error
    - _Requirements: REQ-3_
    - _Dependencies: Task 3.1_

- [ ] 4. Cookie export and import
  
  - [ ] 4.1 Add `cmd_import_cookies` to `CommandDispatcher`
    - Add handler in `lib/browserctl/server/command_dispatcher.rb`
    - Register `"import_cookies"` in the dispatch switch
    - Iterate cookies array, call `session.page.cookies.set(...)` for each
    - _Requirements: REQ-4_
  
  - [ ] 4.2 Add `export_cookies` and `import_cookies` to `Client`
    - Add both methods to `lib/browserctl/client.rb`
    - `export_cookies`: calls existing `cmd_cookies`, writes JSON to file
    - `import_cookies`: reads JSON file, calls new `cmd_import_cookies`
    - _Requirements: REQ-4_
    - _Dependencies: Task 4.1_
  
  - [ ] 4.3 Add CLI subcommands
    - Create `lib/browserctl/commands/export_cookies.rb`
    - Create `lib/browserctl/commands/import_cookies.rb`
    - Wire both into `bin/browserctl` command dispatcher
    - _Requirements: REQ-4_
    - _Dependencies: Task 4.2_
  
  - [ ] 4.4 Wire `.browserctl/sessions/` into `init` and `.gitignore`
    - Add `sessions/` to the `.browserctl/.gitignore` generated by `lib/browserctl/commands/init.rb`
    - Add note to README and `docs/writing-workflows.md` explaining project-scoped session files
    - _Requirements: REQ-4_

- [ ] 5. Params file loading
  
  - [ ] 5.1 Add `Runner.load_params_file`
    - Add class method to `lib/browserctl/runner.rb`
    - Support `.yml`, `.yaml`, `.json` extensions via `YAML.safe_load_file` / `JSON.parse`
    - Rescue `Psych::SyntaxError` and `JSON::ParserError` with clear messages
    - _Requirements: REQ-5_
  
  - [ ] 5.2 Wire `--params` flag into `run` CLI command
    - Add `--params PATH` flag to `lib/browserctl/commands/run.rb` via Optimist
    - Load file params, merge with CLI params (CLI wins), pass merged hash to `Runner.run_workflow`
    - _Requirements: REQ-5_
    - _Dependencies: Task 5.1_
  
  - [ ] 5.3 Add spec coverage
    - Add tests to `spec/unit/runner_spec.rb` for params file loading, merge order, error cases
    - _Requirements: REQ-5_
    - _Dependencies: Task 5.1_

- [ ] 6. Filter token-like query params from recorded goto URLs
  
  - [ ] 6.1 Add `redact_url` to `Recording`
    - Add private class method `Recording.redact_url(url)` in `lib/browserctl/recording.rb`
    - Strip/replace query params matching `SENSITIVE_PARAM_PATTERN` with `[REDACTED]`
    - Rescue `URI::InvalidURIError` — return original URL on parse failure
    - _Requirements: REQ-6_
  
  - [ ] 6.2 Apply redaction in `Recording.append`
    - Call `redact_url` on `:url` when `cmd` is `"goto"` or `"open_page"` before writing to JSONL
    - _Requirements: REQ-6_
    - _Dependencies: Task 6.1_
  
  - [ ] 6.3 Add redaction comment in generated workflow steps
    - In `Recording.build_step`, detect if URL contains `[REDACTED]` and prepend a comment line
    - _Requirements: REQ-6_
    - _Dependencies: Task 6.2_
  
  - [ ] 6.4 Add spec coverage
    - Add tests to `spec/unit/recording_spec.rb` for redaction happy path, no-op on clean URLs, fallback on malformed URLs
    - _Requirements: REQ-6_
    - _Dependencies: Task 6.1_
